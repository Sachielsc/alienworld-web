### Frontend notes
- Vue Router in HTML5 history mode reproduces current URLs (`/about/cv`, `/about/worklog`, ...) so old links keep working.
- Keep the current visual identity (dark alien theme, `#97ff77` green, Inconsolata/Orbitron fonts, egg/head secret menus) but rebuild the CSS cleanly; upgrade to Bootstrap 5 (npm dependency, not vendored) or plain CSS grid — decided during implementation, whichever keeps the theme CSS smaller.
- Google Fonts become self-hosted files (no third-party CDN dependency).
- All existing content pages (work reports m1–m19, study notes, articles, movies/games, CV) are migrated as-is; polishing their inner content is out of scope for this pass.

### Auth design (updated: basic user accounts)
- **Database**: SQLite via `better-sqlite3` — single file (`server/data/alienworld.db`, volume-mounted in Docker), no extra database container.
- **Tables**: `users` (id, username, password_hash, role `admin|member`, created_at) + persistent session store.
- **Endpoints**: `POST /api/register`, `POST /api/login`, `POST /api/logout`, `GET /api/me`. Passwords hashed with bcrypt; `express-session` HTTP-only cookie; `express-rate-limit` on register/login.
- **Roles**: the admin account is seeded on first boot from `ADMIN_USERNAME`/`ADMIN_PASSWORD` env vars. Cover letter & work log content is **admin-only** (they're personal documents — any stranger can register a member account). Member accounts are the foundation for future Community Hub features.
- Cover letter & work log pages fetch content from `/api/protected/...`; server returns 401/403 until an authorized session exists. Content files live outside the static root.

## Execution phases

### Phase 1 — Scaffold (local dev)
1. Create `client/` (Vite + Vue 3 + Vue Router) and `server/` (Express, helmet, express-session, bcrypt, rate-limit).
2. Root `package.json` with npm workspaces + scripts: `npm run dev` (Vite + server concurrently), `npm run build`, `npm start`.
3. Move current app to git history only (delete `public/` after migration completes — it stays recoverable via git).

### Phase 2 — Migrate pages
1. Rebuild layout shell: header, title, egg-menu nav, head-icon secret nav, preloader, theme CSS.
2. Port each module: home, about (statepanel, skillpanel, cv, contactme, otherprojects, workreport), community + articles, movies, games, report (notes + work reports + code snippets).
3. Cover letter generator form (intern/junior/recruitment prologue logic) becomes a Vue component.

### Phase 3 — Auth
1. SQLite setup (users table, seeded admin, persistent sessions).
2. Implement `/api/register|login|logout|me` + protected content routes; move cover letter & work log HTML into `server/protected-content/`.
3. Register/login UI + login gate on the two protected pages; remove every trace of the old hardcoded password. **The old password is burned (it's in git history) — choose a new one.**

### Phase 4 — Local verification
- Full click-through of every route in dev, plus: direct-URL fetch of protected content while logged out must 401; wrong password rejected; refresh keeps session; deep links work.

### Phase 5 — Containerize
1. Multi-stage `Dockerfile`: `node:22-alpine` build of client → runtime image with server + `client/dist` only.
2. `docker-compose.yml`: `app` + `nginx` (ports 80/443) + certbot volume for Let's Encrypt.
3. Verify the full stack locally with `docker compose up` before touching the CVM.

### Phase 6 — Deploy to Tencent CVM (instructions delivered, run together)
1. DNS: A record for the domain → CVM public IP.
2. CVM prep: install Docker + compose plugin; open ports 80/443 in the Tencent security group.
3. Ship code (git clone on CVM, or build image locally and push to a registry — TBC below).
4. Create `.env` on the server (new password hash, session secret), `docker compose up -d`, issue Let's Encrypt cert, verify HTTPS + auth in production.
