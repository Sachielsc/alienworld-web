A problem about sudo apt-get update:

This problem is found by my failure to run sudo apt-get update

Every time I attempt to perform a software update I get this error from bash: 

The repository 'http://ppa.launchpad.net/ubuntu-vn/ppa/ubuntu xenial Release' does not have a Release file.

问题所在：
https://askubuntu.com/questions/768569/ubuntu-16-04-update-manager-error
You obviously did a system upgrade without having removed the external repositories before. The easiest way to solve the problem would be to remove all external repositories and delete all related public keys. After having done this, update the repositories (sudo apt-get update) and check whether everything is fixed now ... and when no error occurs anymore, you can add new external repositories, but carefully check whether these repositories are "matching Xenial" ...

解决方式：
https://askubuntu.com/questions/307/how-can-ppas-be-removed 

Last but not least, you can also disable or remove PPAs from the "Software Sources" section in Ubuntu Settings with a few clicks of your mouse (no terminal needed).

Create a short cut of a folder in ubuntu: (probably not a very good way to do)

sudo ln -sT /path/to/source/directory /path/to/destination/directory

[my example]

sudo ln -sT ~/testproject/alienworld-web/ ~/Desktop/alienworld-web

Create soft link / symbolic link:

sudo ln -s /path/to/source/file /path/to/destination/link-name

Reference: https://www.cyberciti.biz/faq/creating-soft-link-or-symbolic-link/
